## ✅ 완료한 작업

### 1. 옵션 B — 인천·경기 행정동 단위 상권 폴리곤 빌드

- **데이터 출처**: `vuski/admdongkor` GitHub (행정동 GeoJSON, 34MB) — 라이선스 자유
- **빌드 스크립트 신설**: `app/scripts/build-commercial-zones-by-dong.mjs`
  - 인천(sido='28') + 경기(sido='41') = 760 행정동 필터
  - 각 행정동 centroid → 카카오 FD6+CE7 점포 카운트
  - 임계 통과 행정동만 zones.geojson에 union
- **1차 빌드 (임계 30)**: 통과 409 / skip 351 (5분 7초). 단지 매칭 0건 발견 (외곽 신도시 점포 적어 skip)
- **2차 빌드 (임계 10)**: (진행 중 / 결과는 다른 섹션 참조)

### 2. `commercial-zones-official.ts` 알고리즘 보강

- **point-in-polygon 추가**: ray-casting 알고리즘. Polygon + MultiPolygon 모두 처리
- 단지 점이 폴리곤 안이면 `distanceM=0` 무조건 1순위 매칭 (centroid 거리 무관)
- 행정동은 큰 영역이라 centroid가 단지에서 3~5km 떨어져도 단지는 그 안일 수 있음
- centroid radius 1500m → 3000m 확대 (행정동 단위 cover)

### 3. `LocationSection.tsx` 변수명 정리
- `sbaZones` → `officialZones` (이제 SBA + 행정동 통합이라 명확)
- 카카오 동적 fallback 코드 그대로 유지 (시드 안 된 광역만 호출)

### 4. 백로그 26차 세션 기록 추가
