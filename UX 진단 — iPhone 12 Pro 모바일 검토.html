한 줄 요약

모바일에서 화면의 절반 이상을 지도가 점유하고, 네이티브 공유 버튼이 모바일에서 숨겨져 있어 카톡 공유 채널이 막힘이 가장 큰 회복 포인트. 나머지는 폰트 사이즈·그리드 분기·터치 타겟 등 디테일 정정이며, 한 번 정리하면 모바일 가독성이 한 단계 올라간다.

검토 한계

/report/[id]는 세션 인증이 필요해 라이브 렌더링 화면을 직접 캡처하지 못했다. 본 검토는 13개 컴포넌트 소스 코드와 Tailwind 클래스의 모바일(<640px) 분기를 정밀 분석한 결과이며, Recharts 차트의 X축 tick 겹침·카카오맵 마커 충돌·framer-motion 애니메이션 끊김 등 동적 동작은 실 디바이스 또는 Chrome DevTools 디바이스 모드 후속 QA가 권장된다.

Critical 1. 즉시 수정

1.1 ShareBar 네이티브 공유 버튼이 모바일에서 숨겨져 있다 (역설)

위치: src/components/report/ShareBar.tsx:81-87

<button onClick={handleNativeShare}
  className="hidden ... sm:flex">  ← sm 이상에서만 보임
  <Share2 /> 공유
</button>
  • handleNativeShare는 navigator.share를 호출하는데 이 API는 모바일(iOS Safari·Android)에서만 의미가 있다.
  • 데스크탑은 native share가 어색해 안 쓰는 게 맞는데, 현재는 정반대 — 모바일에서 버튼이 숨겨져 있고 데스크탑에서만 보인다.
  • 결과: 카카오톡·메시지 앱 같은 모바일 핵심 공유 채널이 모두 막힘. 990원 결제 후 가족에게 카톡 공유로 보여주는 자연스러운 바이럴 시나리오가 죽음.

해결: hidden sm:flex → flex (또는 환경 감지로 분기). 단일 버튼이 모바일에서 native share, 데스크탑에서 fallback copy로 자연 분기되게.

1.2 LifeScenario — 모바일에서 4 카드가 세로 1열, 본문 11px

위치: src/components/report/LifeScenario.tsx:223

className="mt-5 grid auto-rows-fr gap-3 break-keep sm:grid-cols-2 lg:grid-cols-4"
  • 베이스 columns 미정의 → 모바일 = 1열. 4개 카드가 세로로 쌓여 스크롤 길이 폭증.
  • 카드 본문 text-[11px] → iPhone 12 Pro에서 권장 최소(14px) 미달. 한국어는 자형이 복잡해 더 작게 느껴짐.
  • WhyChillaeMallaeSection의 동일 미리보기는 grid-cols-2 sm:grid-cols-4로 모바일에서 2열. 랜딩 미리보기와 실제 리포트의 그리드가 일치하지 않음 → 사용자 기대치 깨짐.

해결: 베이스 grid-cols-2 명시 + 본문 text-xs(12px) 이상 상향.

1.3 NeighborhoodMap — 화면의 54%가 지도

위치: src/components/report/NeighborhoodMap.tsx:178

className="map-frame-wrap relative h-[460px] w-full"
  • iPhone 12 Pro 뷰포트 844px 중 460px = 54%를 지도가 점유.
  • 좌측 하단 범례 max-w-[340px] → 390px 화면에서 폭 거의 전부, 지도 좌측 절반을 가림.
  • 마커 라벨이 학교 4-5개 + 주변단지 5개 + 폴리곤 라벨이면 가독성 급락 (anti-collision 로직 없음).
  • 줌 컨트롤 미장착 → 핀치 줌만 가능. 한 손 사용 시 어렵다.

해결

  1. 모바일 높이 h-[360px] 또는 h-[50vh]로 축소
  2. 범례를 우상단 토글 버튼(범례 ▼)으로 접어두기
  3. 카카오맵 줌 컨트롤 활성화 (new kakao.maps.ZoomControl())

1.4 ReportByline — 모바일 2×2 그리드에서 SUBJECT 잘림

위치: src/components/report/ReportByline.tsx:50

className="grid grid-cols-2 ... sm:grid-cols-4"
  • 4개 셀(SUBJECT/PERSPECTIVE/PRIORITY/DATE) → 모바일 2×2.
  • SUBJECT가 비교 리포트일 때 "여의도삼부 vs 시범아파트" — 셀 폭 ≈ 165px에 14px 폰트로 들어가면 break-keep 단어 단위 줄바꿈으로 어색.

해결: 모바일은 grid-cols-1 sm:grid-cols-4 또는 SUBJECT만 col-span-2.

High 2. 전환·가독성 직격

2.1 HookHighlights Hero — 우측 destination 블록 압축

위치: src/components/report/HookHighlights.tsx:296-336

<div className="mt-4 flex items-end gap-3">
  <div className="min-w-0">{heroMain}</div>     ← text-5xl (48px)
  <div className="ml-auto shrink-0 text-right">{heroRight}</div>
</div>
  • 좌측 48px + 우측 24px 한 줄. 우측 shrink-0이라 좌측만 압축됨.
  • 상단 heroTopRight(예: "여의나루역 5호선 · 320m")는 truncate text-xs → 모바일에서 ellipsis 잘림 빈번.

해결: 모바일에서 좌·우를 세로 적층 (flex-col sm:flex-row).

2.2 InsightCards — 모바일 1열에서 6 카드, 1순위 ★ 인지 약함

위치: src/components/report/InsightCards.tsx:226

  • 모바일 1열 × 6 = 매우 긴 세로 스택.
  • 1순위 ring-2로 강조하지만 모바일에서 1순위에서 다른 카드로 가려면 5번 스크롤 → 차별화 임팩트 약화.

해결: 모바일 1순위 카드는 border-2 + 헤드라인 text-xl 상향. 또는 2~6순위를 처음 3개만 노출 + "더 보기" 토글.

2.3 TradeFlowTabs — 표 + ▲최고 pill이 거래가 옆에 붙어 폭 부족

위치: src/components/report/TradeFlowTabs.tsx:177-202

  • 모바일 폭 ≈ 342px에서 4컬럼(거래월/거래가/평당가/층) + ▲최고 pill 압축.
  • overflow-x-auto는 있지만 첫 진입 시 한눈에 안 보이는 게 문제.
  • PriceChart h-56(224px) — 18건 거래 시 X축 tick 겹침 (interval 미설정).

해결: 모바일은 표 → 카드 리스트 변환(거래월 / 거래가(평당가)). 차트는 interval="preserveStartEnd".

2.4 NeighborhoodMap 범례 토글 — 터치 타겟 작음

위치: src/components/report/NeighborhoodMap.tsx:209-249

  • 버튼 패딩 없음, h-3(12px) 색칩 + 텍스트만. 실 터치 타겟 ≈ 24px.
  • Apple HIG 권장 44×44pt 미달, 다섯 토글이 gap-y-1.5로 매우 가까이 → 잘못 누르기 쉬움.

해결: py-1.5 px-2 -mx-2로 클릭 영역 ≥40px 확보.

2.5 헤더 제목 — 모바일에서 가용 폭 322px

위치: src/app/report/[id]/page.tsx:114

<h1 className="mt-4 border-l-4 border-primary pl-5 text-3xl ... sm:text-4xl">
  {report.title}
</h1>
  • 모바일: 30px + pl-5(20px) + border 4px → 가용 폭 ≈ 322px. 비교 리포트 긴 제목은 4-5줄로 깨짐.
  • 30px × 5줄 = 150px 점유 → first paint에서 핵심 콘텐츠가 너무 아래로 밀림.

해결: 모바일 text-2xl 다운, sm:text-3xl. 또는 text-balance로 자동 균형.

Medium 3. 일관성·디테일

3.1 폰트 사이즈 일괄 점검 — 11px 미만이 너무 많다

iPhone 12 Pro 권장 최소 12px / 본문 14~16px 기준:

컴포넌트위치사이즈평가

해결: text-[10px] → text-[11px] 또는 text-xs(12px) 일괄 상향. 본문은 13px 이상 권장.

3.2 RegionPercentileBar — wrap 시 시각 흐트러짐

위치: src/components/report/RegionPercentileBar.tsx:25-46

3개 노드(스코프·평당가·차이) + 아이콘이 flex-wrap으로 모바일 폭에서 wrap. 한 줄 안 들어가면 2-3줄로 분리되며 시선 흐름 깨짐.

해결: 모바일 flex-col 강제, sm:flex-row. 또는 가장 중요한 노드(평균 대비 %)만 노출.

3.3 CompareSuggestionsCard CTA 줄바꿈

위치: src/components/report/CompareSuggestionsCard.tsx:88-93

{currentApartmentName} + 추천 단지 {suggestions.length}곳 나란히 보기 · 990원
  • 모바일 폭 ≈ 290px에 14px 폰트 → 줄바꿈 빈번.
  • "추천 단지" 카피 자체가 1차 리포트 톤 가이드 위반 지적과 겹침 — 합쳐서 수정.

해결: 카피 단축 → 옆 단지랑 나란히 보기 · 990원.

3.4 ApartmentSpecs — 모바일 1열에서 'A/B' 라벨 의미 약화

위치: src/components/report/ApartmentSpecs.tsx:125, 190

  • sm:grid-cols-3 + sm:grid-cols-2 → 모바일 1열, 카드 5개 누적.
  • A/B 라벨이 좌우 비교 의도였는데 모바일은 위아래로 쌓이니 'A','B' 의미가 약해짐.

해결: 모바일에서 A/B 대신 평형 자체를 강조 (전용 84㎡ (대표 평형 1)).

3.5 ProfileBadge "관점 바꿔서 다시 보기 →" — 클릭 어포던스 약함

위치: src/components/report/ProfileBadge.tsx:109-114

className="ml-auto rounded-full px-2 py-0.5 text-[11px] text-primary-ink hover:underline"

11px + 패딩 거의 없음 + hover:underline → 모바일은 hover 없어 평소엔 밑줄도 없음. 링크인지 텍스트인지 시각적 구분 약함.

해결: 모바일에서 항상 underline 또는 화살표 아이콘 + 약한 배경(bg-primary-soft)으로 버튼화.

3.6 sticky 진행 바 위치 — Header(top-14)와의 갭 0

위치: src/components/report/ReportTocBar.tsx:64-67

Header 바로 밑에 붙음. 다크모드에서 2px 라인이 헤더와 분리 안 됨.

해결: 모바일은 Header bottom-border 자체를 진행 바로 활용. 또는 h-[3px] + bg-foreground/5 트랙.

6. 점검 한계 및 후속 검증 권장

  • 인증 세션 차단으로 라이브 화면을 직접 못 봄 — Recharts 차트의 모바일 X축 tick 겹침, 카카오맵 마커 anti-collision, framer-motion 애니메이션 끊김 등은 실 디바이스 QA가 추가로 필요.
  • 점검은 13개 리포트 컴포넌트 코드 직독 + Tailwind 클래스 모바일 분기 분석 기반.
  • 후속 검증: 실 iPhone 12 Pro 또는 Chrome DevTools Device Toolbar > iPhone 12 Pro 모드에서 위 시나리오로 한 번 더 확인.
