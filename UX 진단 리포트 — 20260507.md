## 1. 🔴 Critical — 즉시 고쳐야 할 것

### 1.1 모바일에서 헤더 네비게이션이 사라진다

**위치**: `src/components/layout/Header.tsx:19-23`

```tsx
<Link href="/analyze" className="hidden hover:text-foreground sm:inline">
  펼쳐보기
</Link>
<Link href="/mypage" className="hidden hover:text-foreground sm:inline">
  보관함
</Link>
```

`hidden ... sm:inline` 때문에 **모바일(<640px)에서는 두 링크가 모두 숨겨진다.** 햄버거 메뉴도 없다.

- 모바일에서 보관함 들어가는 길 = 마이페이지 직링크 외에 사실상 없음
- 인증된 사용자가 자기 리포트 다시 보려면 URL을 외워야 함
- CLAUDE.md L440 "모바일 퍼스트" 원칙과 정면 충돌

**해결**:
- 모바일에서도 두 링크 노출 (아이콘 only로 작게)
- 또는 우상단 햄버거 메뉴 추가
- AuthNav 옆에 `<Link>`만 살짝 보이게 해도 충분

### 1.2 무료 1회 소진 후, 사용자가 "어떤 단지로 무료를 썼는지" 안 보인다

**위치**: `src/app/analyze/page.tsx:346-372`

무료 쿼터 소진 시 안내 박스가 나오지만 카피가 다음과 같다:

> "이 단지로 받은 무료 리포트가 있어요. 바로 열어볼까요?"

- 첫 진입 사용자는 **자기가 어떤 단지로 무료를 썼는지 자주 잊어버린다**
- "이전 리포트 열기" 버튼이 있지만 단지명이 노출 안 됨
- 다른 단지를 검색하다 무료가 이미 써진 걸 모르고 무료 분석을 시도하면 → "어디서 썼지?" 멘붕

**해결**: 박스 상단에 `📍 [단지명] 으로 펼쳤어요` 한 줄 추가. `existingReportId`로 단지명 join해서 노출. 0.5일 작업.

### 1.3 데이터 신뢰 붕괴 케이스 — "가까운 역" 부정확

라이브 API(`/api/apartments/search`) 직접 호출 결과:

| 단지 | API가 반환한 가까운 역 | 문제 |
|------|------------------------|------|
| 여의도 금호리첸시아 (여의도동 61) | 노량진역 1호선 549m | 여의도 한복판인데 한강 건너 노량진을 추천 — 여의나루역(5호선)이 더 가까울 가능성 |
| 디아크리온강남 (자곡동) | 수서역 GTX-A 1174m | **미개통 GTX**를 nearest로 표기, 거리도 1km 초과 |
| 강남엘에이치 1·2·3단지, 신동아파밀리에 1·2·3단지 | null | "가까운 역 정보 없음" 처리 없이 그냥 비어있음 |
| 헬리오시티 (9510세대) | 석촌고분역 9호선 757m | 송파나루역/잠실역이 더 메인 동선임에도 한 곳만 표기 |

**심각도**: CLAUDE.md L18 "데이터 환각 금지"의 핵심 원칙. AI가 환각 안 한다고 강조해도 **DB 원본이 오류면 의미 없음.** 사용자가 "이거 좀 이상한데?" 한 번 느끼면 990원 결제 의사가 0이 된다.

**해결**:
- nearestStation이 null인 단지는 검색 카드에서 칩 자체를 숨김
- GTX 미개통 노선은 표기에 `(예정)` 명시 또는 후순위 처리
- 거리 1km 초과 시 "도보 + 마을버스" 톤으로 다운그레이드
- (백로그) 카카오 keyword API 좌표 vs DB nearest 검증 스크립트 추가

## 2. 🟠 High — 전환율을 직접 깎고 있는 것

### 2.1 검색 동명 단지 구분 불가

`래미안` 검색 시 동명 단지 2건 동시 노출:
- `id=0de7d8a5...` 송파구 문정동 1
- `id=eda6ec05...` 서초구 서초동 1682

둘 다 단지명이 그냥 "래미안". 모바일에서는 주소가 잘려보일 수 있어 사용자가 어느 게 어디인지 즉시 구분 어려움.

**해결**:
- 동명 단지에 자동으로 자치구 prefix (`송파구 래미안 / 서초구 래미안`)
- 주소 ellipsis 시 동(洞) 정보까지는 우선 표시되게 폭 조정

### 2.2 ProfileForm 3-step의 "30초"는 과장

**위치**: `src/components/analyze/ProfileForm.tsx`

7개 가구형태 → 8개 우선순위(최대 3개) → 6개 출근지. 옵션 수 대비 **최소 30초**, 평균 1분 이상.

추가 마찰:
- 헤더에 "STEP · 3단계 · 약 30초" 배지 (`analyze/page.tsx:237-239`) 있는데, 각 step 안에는 진행도 N/3 외에 "지금 어디" 표시가 없다
- Step 3 출근지에 **명시적 "건너뛰기" 버튼이 없다.** 카피는 "건너뛰셔도 돼요"라고 말하지만 실제로는 그냥 "한 장 펼쳐보기" 누르는 거지, 명시 SKIP 버튼 부재
- "한 장 펼쳐보기" CTA가 disabled일 때 **왜 disabled인지 안 알려줌** → 사용자 멈춤

**해결**:
1. Step 3에 보조 버튼 `[건너뛰고 바로 펼치기]` 추가
2. 진행도 인디케이터 옆에 `1/3 가족 → 우선순위 → 출근지` 미니 텍스트
3. disabled 버튼에 hover/focus 시 `"가족 형태를 먼저 골라주세요"` 툴팁 또는 상단 hint

### 2.3 첫 사용자 funnel 마찰 너무 많다

```
랜딩 → 검색 → 단지 클릭 → /analyze에서 카드 확인 → "무료로 분석 받기"
   → SMS 인증 모달 (번호 + OTP 6자리, 2 step)
   → 모달 닫힘 → 프로필 폼 진입 (3 step)
   → "한 장 펼쳐보기" → 로딩
   → 리포트
```

**총 7~8 클릭.** 평균 모바일 사용자 이탈률 30%/단계 가정 → **끝까지 가는 비율 ≈ 35%**. CLAUDE.md L17 "진입장벽 제로 — SMS 인증만"이 코어 원칙임에도 실제로는 SMS + 프로필 3-step.

**단계적 해결**:
- (즉시) 인증 모달 안에 "어떤 가구이신가요?" Step1을 함께 받기 → 인증 + 가족형태 한 화면. 우선순위·출근지는 리포트 본 뒤 ProfileBadge에서 "더 정확하게 보려면" 톤으로 점진 유도
- (중기) **첫 무료 리포트는 프로필 없이도 생성**되게 하고, 리포트 내 "관점 추가하기" 카드로 전환
- 이 한 가지가 990원 전환 funnel에서 가장 큰 ROI

### 2.4 결제 성공 페이지에서 사용자가 막힐 수 있다

**위치**: `src/app/payment/success/page.tsx:18-24`

paymentKey/orderId/amount 중 하나라도 없으면 `"결제 정보가 누락되었어요"` 메시지만 표시. **다음 액션 버튼이 없다.** 새로고침/뒤로가기 외에 길이 없음.

**해결**: 에러 상태에 `[고객문의] [/compare로 돌아가기]` 두 버튼 + 안내 문구.

### 2.5 결제 실패 페이지 — 토스 에러 메시지 그대로 노출

**위치**: `src/app/payment/fail/page.tsx:11`

## 3. 🟡 Medium — 일관성·디테일

### 3.1 "옆 단지도 칠래말래?" — 첫 입장 시 빈 슬롯 신호 약함

**위치**: `src/app/compare/page.tsx:194-198`

`picks.length > 0 && picks.length < MAX_COMPARE` 조건이라 **0개일 때는 "한 단지 더 추가" 안내가 안 나온다.** 빈 박스만 있고 검색바와의 시각적 연결이 약함.

**해결**: 빈 상태에서도 슬롯 2개를 점선으로 그려두고 `A 검색 → B 검색` 같은 시각적 가이드.

### 3.2 ShareBar 위치 — 업셀과 우선순위 충돌

**위치**: `src/app/report/[id]/page.tsx:204`

ShareBar가 UpsellCTAs **위에** 있다. 공유 의도가 가장 강한 시점은 리포트를 다 본 직후이지만, 그 자리에서 990원 코어 BM 노출이 더 우선이다.

**해결**: ShareBar를 UpsellCTAs **아래**로 이동, 또는 헤더 옆 작은 아이콘 그룹으로 축소하고 본문 끝은 업셀에 양보.

### 3.3 카피 일관성 — "추천" 단어 잠입 (법적 리스크)

CLAUDE.md L88 "추천 표현 금지"에 위배되는 잠입 카피들:

- `CompareSuggestionsCard.tsx:91` — `[현재 단지] + 추천 단지 N곳 나란히 보기` ❌
- `LifeScenario.tsx:221` — `실제 입주 전에 직접 답사해보시는 걸 추천드려요` ❌

좋은 카피 (참고):
- `CompareSuggestionsCard.tsx:30` — `옆 단지랑 나란히 보면 더 명확해져요` ✅
- `analyze/page.tsx:280` — `단지 하나 고르면 교통·학군·시세까지 한 장에 펼쳐드려요` ✅

**해결**:
- 일괄 치환: `추천 단지 → 비교 좋은 단지`, `추천드려요 → 가보세요/직접 보시는 게 좋아요`

### 3.4 마이페이지 보관함 — 빈 상태 카피 약함

**위치**: `src/app/mypage/page.tsx:91-97`

`"아직 펼친 단지가 없어요. 한 장 펼쳐보기 →"` — 톤은 좋지만 재방문 사용자(인증 됐으나 리포트 0인) 시나리오를 놓치고 있음. 가장 활성화 가능한 잠재 결제자.

### 3.5 검색 결과 없음 안내 — 책임 전가형 카피

### 3.8 LifeScenario "도보 5분"의 정확도 신호 부재

## 7. 부록 — 진단 방법 한계

- 라이브 사이트의 SPA 렌더링은 WebFetch로 직접 못 봤다 (Next.js 클라이언트 hydration 후 콘텐츠를 확인 불가). 인터랙션 끊김, 키보드 포커스, 스크롤 락 등 **체감 인터랙션은 후속 검증 필요**.
- 후속 검증 방안: Playwright/Browser MCP로 동일 시나리오 자동 클릭 테스트, 또는 실 디바이스(iPhone/Android) 수동 QA.
- 본 진단은 정적 분석(코드 + 라이브 API) 기반으로, **명확히 코드에서 확인 가능한 사실**과 **사용자 멘탈 모델 추론**을 분리해 작성했다.

## 8. 좋은 점 — 현재 잘 하고 있는 것 (유지 권장)

진단 위주 리포트지만, 깎으려고 깎는 게 아니라 살릴 부분은 살려야 한다.

1. **카피 톤 — 또래 부동산 서비스 대비 압도적으로 잘 잡혀 있다.** "칠래말래?", "한 번 까봐", "옆 단지도 칠래말래" — 의문형·가벼운 톤이 일관됨. CLAUDE.md L55~93 가이드를 코드에서 충실히 구현.
2. **ProfileBadge / ReportByline의 "당신을 위한 관점" 시그니처** — 같은 단지를 가구별로 다르게 풀어준다는 걸 시각화한 솔루션이 영리하다. 이게 990원 결제의 emotional hook.
3. **HookHighlights의 Hero + Sub 카드 구조** — 1순위는 큰 그라데이션 카드, 나머지는 보조 카드. 정보 위계가 명확.
4. **TradeFlowTabs의 평형 칩 탭 + 최고/최저 강조** — 실거래 표를 한 화면에 정리하면서 시각적 단서가 충분.
5. **무료/유료 경계의 명확함** — `free_deep_single` vs `compare_report` 가격 상수와 서버 검증 로직이 깔끔. 결제 보안의 코어가 잘 잡혀 있다.
6. **다크모드 + 따뜻한 강조 컬러 (rose/sky/amber)** — 부동산의 무거움을 색으로 풀고 있다.
