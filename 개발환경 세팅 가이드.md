# 🛠️ 개발환경 세팅 가이드

> **VS Code + Claude Code Max** 기반으로 입지비교 서비스(입지990)를 개발하기 위한 전체 환경 구성 가이드.
> 기술 스택: **Next.js 14 · Tailwind CSS · Supabase · 토스페이먼츠 · Claude API · Vercel**

---

## 🔰 시작하기 전에 — 이 가이드의 읽는 법

이 가이드에서는 **"어디서 하는 작업인지"**를 아이콘으로 구분합니다:

| 아이콘 | 의미 | 설명 |
|:---:|------|------|
| 🖱️ | **설치 파일 실행** | 웹에서 .exe 파일 다운로드 → 더블클릭 설치 (Windows 일반 프로그램 설치) |
| 🌐 | **웹 브라우저** | 크롬 등 브라우저에서 사이트 접속하여 수행하는 작업 |
| 💻 | **VS Code 내장 터미널** | VS Code 안에 있는 터미널에서 명령어 입력 |
| 🔧 | **VS Code UI** | VS Code의 메뉴, 사이드바, 설정 등에서 마우스로 클릭하여 수행 |

> [!important] 별도 터미널(PowerShell, CMD)을 따로 열 필요 없습니다!
> **VS Code 안에 터미널이 내장**되어 있어서, `npm install` 같은 명령어도 모두 VS Code 안에서 실행합니다.
> 이 가이드의 모든 명령어는 VS Code 내장 터미널에서 실행하는 것입니다.

### VS Code 내장 터미널이란?

VS Code 하단에 있는 터미널 패널입니다. 별도의 PowerShell/CMD 창을 열 필요 없이, **코드 편집과 명령어 실행을 한 화면에서** 할 수 있습니다.

```
┌─────────────────────────────────────────────┐
│  VS Code                                     │
│  ┌───────────────────────────────────────┐   │
│  │                                       │   │
│  │         코드 편집 영역                  │   │
│  │         (여기서 코드를 보고 수정)        │   │
│  │                                       │   │
│  ├───────────────────────────────────────┤   │
│  │  💻 터미널 (하단 패널)                  │   │
│  │  PS C:\Users\LSH> npm install         │   │
│  │  ← 여기서 명령어 입력                   │   │
│  └───────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**터미널 여는 법:**
- 단축키: `` Ctrl+` `` (백틱 키, 숫자 1 왼쪽)
- 또는 상단 메뉴: **터미널 → 새 터미널**
- 또는 상단 메뉴: **보기 → 터미널**

## 2. 필수 프로그램 설치 (Windows 설치파일)

> [!note] 이 섹션은 모두 **🖱️ 설치 파일 실행** 방식입니다.
> 웹사이트에서 .exe 파일을 다운로드 → 더블클릭으로 설치합니다.
> 터미널 명령어가 아닙니다.

### 2.1 🖱️ Node.js 설치

Node.js는 JavaScript를 실행할 수 있게 해주는 프로그램입니다. `npm` (패키지 관리자)이 함께 설치됩니다.

**설치 순서:**

1. 🌐 https://nodejs.org/ko 접속
2. 🖱️ **LTS** 버전 다운로드 (안정 버전, 초록색 버튼)
3. 🖱️ 다운로드된 `.msi` 파일 더블클릭 → **모두 기본값(Next)** 으로 설치
4. 설치 중 **"Automatically install the necessary tools"** 체크박스가 나오면 체크

**설치 확인** (나중에 VS Code 터미널에서):

```
💻 VS Code 내장 터미널에서 실행
node -v    # v20.x 이상이 나오면 성공
npm -v     # 10.x 이상이 나오면 성공
```

> [!tip] nvm-windows 사용 (선택사항, 고급)
> 여러 프로젝트에서 다른 Node 버전이 필요할 때 유용합니다.
> 1. 🌐 https://github.com/coreybutler/nvm-windows/releases 접속
> 2. 🖱️ `nvm-setup.exe` 다운로드 → 설치
> 3. 💻 VS Code 터미널에서: `nvm install lts` → `nvm use lts`

### 2.2 🖱️ Git 설치

Git은 코드 버전 관리 도구입니다. 코드 변경 이력을 기록하고, 실수해도 이전으로 돌아갈 수 있습니다.

**설치 순서:**

1. 🌐 https://git-scm.com/download/win 접속
2. 🖱️ **64-bit Git for Windows Setup** 다운로드
3. 🖱️ 다운로드된 `.exe` 파일 더블클릭 → **모두 기본값(Next)** 으로 설치

**설치 확인 및 초기 설정** (나중에 VS Code 터미널에서):

```
💻 VS Code 내장 터미널에서 실행
git --version                                        # 버전 나오면 성공
git config --global user.name "이름"                  # 본인 이름
git config --global user.email "your@email.com"       # 본인 이메일
git config --global init.defaultBranch main
```

### 2.3 🖱️ (선택) GitHub Desktop 설치

Git 명령어(CLI)가 익숙하지 않다면, GUI로 커밋/푸시를 할 수 있는 GitHub Desktop을 함께 설치하세요.

1. 🌐 https://desktop.github.com/ 접속
2. 🖱️ 다운로드 → 설치

## 4. Claude Code 설치 및 연동

### 4.1 💻 Claude Code CLI 설치 — VS Code 내장 터미널에서

Claude Code를 사용하려면 CLI(명령줄 도구)를 설치해야 합니다.
**VS Code의 내장 터미널**에서 아래 명령어를 실행합니다.

```
💻 VS Code 내장 터미널 열기: Ctrl+` (백틱)

npm install -g @anthropic-ai/claude-code
```

- `npm`은 섹션 2에서 Node.js 설치 시 함께 설치된 패키지 관리자입니다.
- `-g`는 "전역(global) 설치" — 이 컴퓨터 어디서든 `claude` 명령을 쓸 수 있게 합니다.

**설치 확인:**

```
💻 VS Code 내장 터미널에서 실행
claude --version    # 버전 번호가 나오면 성공
```

### 4.2 Extension과 CLI의 관계

```
┌─────────────────────────────────────────────┐
│  VS Code                                     │
│                                              │
│  🔧 Claude Code Extension (사이드바 UI)       │
│     ✦ Spark 아이콘 클릭 → 대화창              │
│     ✦ 코드 선택 → Claude에 전송               │
│     ✦ 인라인 Diff 미리보기                     │
│              │                               │
│              ▼ 내부적으로 연동됨               │
│                                              │
│  💻 Claude Code CLI (터미널 에이전트)           │
│     ✦ 파일 읽기/쓰기/수정                     │
│     ✦ 터미널 명령 자동 실행                    │
│     ✦ 프로젝트 전체 컨텍스트 분석              │
│                                              │
└─────────────────────────────────────────────┘
```

- **Extension** (섹션 3에서 설치): VS Code 사이드바에서 GUI로 Claude와 대화
- **CLI** (이 섹션에서 설치): 터미널에서 `claude` 명령어로 직접 실행
- 둘 다 같은 Max 구독을 사용하며, **어느 쪽으로 사용해도 동일한 결과**

### 4.3 🔧 인증 (최초 1회)

Max 구독자는 별도 API 키 없이, claude.ai 계정으로 인증합니다.

**방법 A — VS Code UI에서 (추천):**

```
🔧 VS Code에서 수행

1. Ctrl+Shift+P 눌러서 "명령 팔레트" 열기
2. "Claude Code: Authenticate" 검색 → 클릭
3. 브라우저가 열리면 claude.ai 계정으로 로그인
4. 인증 완료!
```

**방법 B — 터미널에서:**

```
💻 VS Code 내장 터미널에서 실행

claude
# → 브라우저가 자동으로 열림 → 로그인 → 인증 완료
```

> [!note] Max 구독 인증 vs 서비스용 API 키
> - **Max 구독 인증** (여기서 하는 것): 내가 개발할 때 Claude Code를 쓰기 위한 것
> - **서비스용 API 키** (섹션 6에서 설명): 완성된 서비스가 사용자에게 리포트를 생성해줄 때 사용하는 것
> - 둘은 별개입니다

### 4.4 주요 명령어 (Claude Code 대화 중)

| 명령어 | 설명 |
|--------|------|
| `claude` | 💻 터미널에서 대화 모드 시작 |
| `claude "작업 내용"` | 💻 터미널에서 단발 명령 실행 |
| `/init` | 대화 중: CLAUDE.md 생성 (프로젝트 컨텍스트 파일) |
| `/cost` | 대화 중: 현재 세션 사용량/비용 확인 |
| `/clear` | 대화 중: 대화 컨텍스트 초기화 |
| `/compact` | 대화 중: 긴 대화를 요약하여 토큰 절약 |
| `Ctrl+C` | 현재 작업 중단 |

### 4.5 CLAUDE.md 설정 (프로젝트 컨텍스트)

프로젝트 루트에 `CLAUDE.md` 파일을 만들면, Claude Code가 프로젝트의 맥락을 자동으로 이해합니다.

**생성 방법:**

```
💻 VS Code 내장 터미널에서 프로젝트 폴더로 이동 후

claude
# Claude Code 대화 시작 후
/init
# → CLAUDE.md 파일이 자동 생성됨
```

생성된 CLAUDE.md에 아래 내용을 추가합니다:

```markdown
# 입지990 — 부동산 입지비교 서비스

## 프로젝트 개요
- 부동산 아파트 입지를 AI로 비교 분석하는 유료 콘텐츠 서비스
- 사주990의 소액 콘텐츠 결제 모델을 부동산에 적용

## 기술 스택
- Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS
- Backend: Next.js API Routes
- DB: Supabase (PostgreSQL)
- 인증: Supabase Auth (카카오/네이버 OAuth)
- 결제: 토스페이먼츠 SDK
- AI: Claude API (Anthropic)
- 배포: Vercel

## 코딩 컨벤션
- 한국어 주석 사용
- 컴포넌트 파일명: PascalCase (예: CompareCard.tsx)
- API 라우트: kebab-case (예: api/real-estate/route.ts)
- 상수: UPPER_SNAKE_CASE
- Tailwind CSS 사용, 인라인 스타일 지양

## 주의사항
- .env.local 파일은 절대 커밋하지 않기
- 부동산 데이터는 공공 API에서 조회, AI는 요약/분석만 담당
- 금액 관련 숫자는 반드시 DB/API 원본 데이터 사용 (AI 환각 방지)
```

## 5. 프로젝트 초기 셋업

> [!note] 이 섹션의 모든 명령어는 **💻 VS Code 내장 터미널**에서 실행합니다.
> 터미널 열기: `` Ctrl+` ``

### 5.1 💻 프로젝트 폴더 열기

먼저 VS Code에서 프로젝트 폴더를 엽니다.

### 7.2 환경변수 명명 규칙

| 접두사 | 의미 | 예시 |
|--------|------|------|
| `NEXT_PUBLIC_` | 브라우저에서 접근 가능 (공개됨) | `NEXT_PUBLIC_SUPABASE_URL` |

> [!caution] 보안 키에는 절대 `NEXT_PUBLIC_` 접두사를 붙이지 마세요!

> 브라우저에 노출되면 누구나 복사하여 악용할 수 있습니다.

## 12. Git 워크플로우

### 12.1 💻 GitHub 저장소 연결

```
💻 VS Code 내장 터미널에서 실행

git remote add origin https://github.com/YOUR-USERNAME/ipji990.git
git push -u origin main
```

> [!note] `create-next-app`이 Git 초기화를 자동으로 해줍니다.
> 별도로 `git init` 할 필요 없습니다.

### 12.2 커밋 메시지 컨벤션

```
feat: 새 기능 추가        → feat: 랜딩 페이지 구현
fix: 버그 수정            → fix: 검색바 자동완성 오류 수정
style: 스타일 변경        → style: 비교 카드 레이아웃 개선
refactor: 리팩토링        → refactor: Supabase 클라이언트 분리
docs: 문서 수정           → docs: README 업데이트
chore: 기타               → chore: 패키지 업데이트
```

### 12.3 일상 커밋 — 2가지 방법

**방법 A — 🔧 VS Code UI에서 (마우스 클릭, 추천):**

```
🔧 VS Code에서 수행

1. 왼쪽 사이드바 "소스 제어" 아이콘 클릭 (가지 모양, 또는 Ctrl+Shift+G)
2. 변경된 파일 목록 확인
3. "+" 버튼 클릭하여 스테이징 (또는 "모두 스테이징")
4. 상단 메시지 입력창에 커밋 메시지 작성 (예: "feat: 랜딩 페이지 구현")
5. ✓ 체크 버튼 클릭 (커밋)
6. "⋯" 메뉴 → "Push" 클릭 (GitHub로 업로드)
```

**방법 B — 💻 터미널에서:**

```
💻 VS Code 내장 터미널에서 실행

git add .
git commit -m "feat: 입지 비교 리포트 페이지 구현"
git push
```

> [!tip] Claude Code에게 커밋도 맡기기
> Claude Code 대화에서: `"현재 변경사항을 확인하고 적절한 커밋 메시지로 커밋해줘"`

## 13. 트러블슈팅

### 자주 발생하는 문제

| 문제 | 원인 | 해결 방법 |
|------|------|----------|
| `'node'은(는) 인식할 수 없는 명령입니다` | Node.js 미설치 또는 PATH 미등록 | 🖱️ Node.js 재설치 (섹션 2.1) |
| `'npm'은(는) 인식할 수 없는 명령입니다` | Node.js 미설치 | 🖱️ Node.js 재설치 (섹션 2.1) |
| `'git'은(는) 인식할 수 없는 명령입니다` | Git 미설치 | 🖱️ Git 재설치 (섹션 2.2) |
| `'claude'은(는) 인식할 수 없는 명령입니다` | Claude Code CLI 미설치 | 💻 `npm install -g @anthropic-ai/claude-code` |
| Claude Code 인증 실패 | 구독 만료 or 미인증 | 🌐 [claude.ai](https://claude.ai)에서 구독 상태 확인 |
| `Module not found` 에러 | 패키지 미설치 | 💻 `npm install` 실행 |
| `.env.local` 변수 안 읽힘 | 서버 재시작 필요 | 💻 `Ctrl+C` → `npm run dev` |
| Tailwind 클래스 안 먹힘 | config 경로 문제 | 🔧 `tailwind.config.ts`에서 content 경로 확인 |
| 토스페이먼츠 결제 실패 | 테스트 키 오류 | 🌐 키 앞에 `test_` 접두사 확인 |
| Supabase 연결 안 됨 | URL/키 오타 | 🌐 Supabase 대시보드에서 키 재복사 |
| `rate limit` 에러 | Max 사용량 초과 | 5시간 대기, `/compact`로 절약 |

### 💻 Node.js 설치 후에도 명령어 인식이 안 될 때

VS Code를 **완전히 종료 후 재시작**해야 PATH가 적용됩니다.

```
🔧 VS Code 완전 종료 → 다시 실행 → 터미널(Ctrl+`) 열기

node -v    # 이제 버전이 나오면 성공
```

### 💻 패키지 설치가 꼬였을 때

```
💻 VS Code 내장 터미널에서 실행

Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
npm install
```
